3  Data Frame

Indexación y Manipulación con R Base y dplyr

Author

Samuel Solís Barrón

3.1 Objetivo

Extraer subconjuntos de un data frame de dos maneras distintas: usando la sintaxis de indexación de R base ([ , ], which(), subset()) y usando los verbos del paquete dplyr del ecosistema Tidyverse (filter(), select(), mutate(), arrange(), relocate(), summarise(), etc.).

3.2 Datos utilizados

  • Paquetes
  • Descripción general
  • Código R
  • iris
  • ChickWeight
#| label: cargar-paquetes
#| eval: true   
#| echo: true    

library(dplyr)     # Manipulación de datos (filtrar, agrupar, resumir, etc.)

La base de datos utilizada es iris, incluida en R base. Proviene de las observaciones de Ronald Fisher (1936) sobre el género Iris. La variable categórica (factor) es Species (setosa, versicolor y virginica), con 50 observaciones por especie. Las variables de respuesta son:

  • Sepal.Length — longitud del sépalo (cm).
  • Sepal.Width — ancho del sépalo (cm).
  • Petal.Length — longitud del pétalo (cm).
  • Petal.Width — ancho del pétalo (cm).

URL: https://onlinelibrary.wiley.com/doi/epdf/10.1111/j.1469-1809.1936.tb02137.x

La base de datos ChickWeight, incluida en R base, proviene de un experimento nutricional descrito por Crowder & Hand (1990) en el que se evaluó el efecto de cuatro dietas proteicas sobre el crecimiento de pollitos. La variable de agrupación (factor) es Diet (dietas 1, 2, 3 y 4), con mediciones repetidas por sujeto a lo largo del tiempo. Las variables del conjunto son:

  • weight — peso corporal del pollito (gramos).
  • Time — día de medición (0 a 21).
  • Chick — identificador del pollito (1 al 50).
  • Diet — dieta proteica asignada (1, 2, 3 ó 4).

El código para cargar las bases de datos es el siguiente:

#| label: data-iris-chick
#| echo: true
#| eval: false

data(iris)            # Carga iris desde R base. No requiere paquetes adicionales.
iris1 <- iris[, 1:4]  # Selecciona solo las 4 columnas numéricas (excluye Species).

data(ChickWeight)     # Carga ChickWeight desde R base.
cw <- ChickWeight     # Asigna un nombre más corto para facilitar su uso en el código.
#| label: tabla-iris
#| echo: false
#| eval: true

data(iris)
iris1 <- iris[, 1:4]
DT::datatable(iris1)
#| label: tabla-chickweight
#| echo: false
#| eval: true

data(ChickWeight)
cw <- ChickWeight
DT::datatable(cw, options = list(pageLength = 10))

3.3.2 R Base: Indexación de Data Frames

La sintaxis general de indexación en R es data[filas, columnas]. Dejar un espacio en blanco en alguna dimensión significa conservar todo en esa dimensión. El operador [ ] es el indexador fundamental de R para vectores, matrices y data frames; para data frames acepta dos argumentos separados por una coma, y omitir uno de los dos conserva esa dimensión completa.


#| label: sintaxis-general
#| echo: true
#| eval: false

datos[filas, columnas]   # seleccionar filas y columnas específicas
datos[ , ]               # conservar todas las filas y todas las columnas

3.3.2.1 Selección de columnas

Las columnas de un data frame pueden seleccionarse por posición (número entero), por nombre (cadena de texto) o excluyéndolas con un índice negativo. El operador : genera una secuencia de enteros consecutivos, y c() construye vectores de índices no contiguos.


#| label: seleccion-columnas
#| echo: true
#| eval: false

# ── Por posición individual ───────────────────────────────────────────────────

nuevo_dato <- datos[ , 40]

# ── Por posiciones no contiguas ───────────────────────────────────────────────

nuevo_dato <- datos[ , c(2, 5)]

# ── Por rango de posiciones contiguas ─────────────────────────────────────────

nuevo_dato <- datos[ , c(2:5)]

# ── Exclusión de una columna (signo negativo) ─────────────────────────────────

nuevo_dato <- datos[ , -5]

3.3.2.2 Selección de filas

Las filas pueden seleccionarse por rango de posición, por condición lógica usando which(), o con la función subset(). which() devuelve los índices numéricos de los elementos de un vector lógico que son TRUE; subset() ofrece una sintaxis más legible para filtrar y seleccionar simultáneamente.


#| label: seleccion-filas
#| echo: true
#| eval: false

# ── Por rango de posición ─────────────────────────────────────────────────────

nuevo_dato <- datos[1:50, ]

# ── Por condición de texto con which() ───────────────────────────────────────

nuevo_dato <- datos[which(datos$gender == "mujer"), ]

# ── Condición combinada con AND lógico (&) ────────────────────────────────────

nuevo_dato <- datos[which(datos$gender == "mujer" & datos$age >= 65), ]

# ── Con subset(): filtrado de filas y selección de columnas en un paso ────────

nuevo_dato <- subset(datos, age >= 20, select = c(ID, Weight))

3.3.3 Tidyverse: Manipulación con dplyr

dplyr es un paquete del ecosistema Tidyverse que ofrece verbos especializados para manipular data frames de forma legible. Las operaciones se encadenan con el operador pipe %>%, que pasa el resultado de una función como primer argumento de la siguiente función en la cadena. La instalación solo es necesaria una vez por entorno de trabajo; la carga con library() debe hacerse al inicio de cada sesión.

3.3.3.1 Verbos principales de dplyr

Verbo Propósito
select() Elegir columnas
filter() Conservar filas según una condición
mutate() Crear o modificar columnas
arrange() Ordenar filas
relocate() Reordenar columnas
summarise() Calcular resúmenes estadísticos
group_by() Agrupar para operaciones por grupo

Se crea el objeto df como copia de iris para trabajar con los ejemplos sin modificar el data frame original.


#| label: crear-df
#| echo: true
#| eval: false

df <- iris

3.3.3.2 select()

select() conserva únicamente las columnas especificadas. El signo negativo excluye una columna por nombre. starts_with() es una función auxiliar (selection helper) que selecciona todas las columnas cuyo nombre comienza con el prefijo indicado, lo que es útil cuando el data frame tiene muchas columnas con un patrón de nombrado común.


#| label: select-ejemplos
#| echo: true
#| eval: false

# ── Por nombre ────────────────────────────────────────────────────────────────

df %>% select(Sepal.Length, Sepal.Width)

# ── Exclusión por nombre ──────────────────────────────────────────────────────

df %>% select(-Species)

# ── Por prefijo del nombre ────────────────────────────────────────────────────

df %>% select(starts_with("Sepal"))

3.3.3.3 filter()

filter() conserva únicamente las filas que cumplen la condición dada. == es el operador de igualdad (dos signos =; uno solo es asignación). & combina condiciones con AND lógico (ambas deben cumplirse), y | las combina con OR (basta con que se cumpla una).


#| label: filter-ejemplos
#| echo: true
#| eval: false

# ── Igualdad de categoría ─────────────────────────────────────────────────────

df %>% filter(Species == "setosa")

# ── Condición numérica ────────────────────────────────────────────────────────

df %>% filter(Sepal.Length > 5)

# ── AND lógico (&) ────────────────────────────────────────────────────────────

df %>% filter(Species == "setosa" & Sepal.Length > 5)

# ── OR lógico (|) ─────────────────────────────────────────────────────────────

df %>% filter(Species == "setosa" | Species == "virginica")

3.3.3.4 mutate()

mutate() añade una nueva columna al data frame sin eliminar las existentes. El nombre de la nueva columna se define a la izquierda del = y la expresión de la derecha puede usar cualquier operación vectorizada de R.


#| label: mutate-ejemplo
#| echo: true
#| eval: false

df %>% mutate(Sepal.Area = Sepal.Length * Sepal.Width)

3.3.3.5 arrange()

arrange() ordena las filas del data frame según la columna indicada. Por defecto el orden es ascendente; desc() invierte el orden a descendente y es una función auxiliar usada exclusivamente dentro de arrange().


#| label: arrange-ejemplo
#| echo: true
#| eval: false

# ── Orden ascendente (por defecto) ────────────────────────────────────────────

df %>% arrange(Sepal.Length)

# ── Orden descendente ─────────────────────────────────────────────────────────

df %>% arrange(desc(Sepal.Length))

3.3.3.6 relocate()

relocate() mueve una o más columnas a una posición distinta del data frame. Sin argumentos adicionales, mueve la columna al principio. El argumento after = la coloca inmediatamente después de la columna de referencia indicada.


#| label: relocate-ejemplo
#| echo: true
#| eval: false

# ── Mover al principio ────────────────────────────────────────────────────────

df %>% relocate(Sepal.Length)

# ── Mover después de una columna específica ───────────────────────────────────

df %>% relocate(Sepal.Length, after = Species)

3.3.3.7 summarise()

summarise() colapsa todo el data frame en una sola fila que contiene las estadísticas calculadas. mean() calcula el promedio aritmético de un vector numérico. El resultado es un data frame de 1 fila × 1 columna.


#| label: summarise-ejemplo
#| echo: true
#| eval: false

df %>% summarise(media_largo = mean(Sepal.Length))

3.3.3.8 group_by() + summarise()

group_by() no modifica los datos visualmente, pero añade metadata interna que indica a summarise() que las operaciones deben realizarse de forma independiente dentro de cada nivel del factor. El resultado es un data frame de tantas filas como grupos existan.


#| label: group-summarise
#| echo: true
#| eval: false

df %>%
  group_by(Species) %>%
  summarise(media_largo = mean(Sepal.Length))

3.3.3.9 Pipeline encadenado — ejemplo completo

Este pipeline combina tres verbos en secuencia:

  1. filter(): conserva solo las flores de la especie setosa.
  2. mutate(): añade la columna Sepal.Area = largo × ancho del sépalo.
  3. group_by() + summarise(): calculan la media del área para esa especie.

El resultado es un data frame de 1 fila × 2 columnas (Species, area_media). El operador %>% evita guardar objetos intermedios y hace el flujo legible de arriba hacia abajo, como una receta de pasos secuenciales.


#| label: pipeline-encadenado
#| echo: true
#| eval: false

df %>%
  filter(Species == "setosa") %>%
  mutate(Sepal.Area = Sepal.Length * Sepal.Width) %>%
  group_by(Species) %>%
  summarise(area_media = mean(Sepal.Area))

3.4 Resultados

3.4.1 R Base con ChickWeight

#| label: res-chick1
#| echo: false
#| eval: true
#| 
chick1 <- ChickWeight[which(ChickWeight$Diet == 2), ]
head(chick1, 5)
    weight Time Chick Diet
221     40    0    21    2
222     50    2    21    2
223     62    4    21    2
224     86    6    21    2
225    125    8    21    2

Interpretación: El subconjunto chick1 contiene todas las observaciones de los pollitos asignados a la dieta 2, conservando íntegramente las cuatro columnas originales (weight, Time, Chick, Diet). La columna Diet toma el valor 2 en todas las filas, confirmando que el filtro se aplicó correctamente. Las primeras filas corresponden a mediciones tempranas (días 0 y 2), donde los pesos son bajos porque los pollitos apenas comenzaban el experimento. which() garantiza que las filas con valores NA en la condición se descarten automáticamente, a diferencia del indexado lógico directo que las conservaría como NA.

#| label: res-chick2
#| echo: false
#| eval: true

chick2 <- ChickWeight[ChickWeight$Chick == 4, c("weight", "Time")]
head(chick2, 5)
   weight Time
37     42    0
38     49    2
39     56    4
40     67    6
41     74    8

Interpretación: chick2 contiene el historial de peso del pollito número 4 a lo largo del tiempo, mostrando únicamente las columnas weight y Time. Esta estructura bidimensional es ideal para graficar la curva de crecimiento individual: Time en el eje X y weight en el eje Y. La selección de columnas por nombre es más robusta que por posición numérica, ya que no depende del orden de las columnas en el data frame original.

#| label: res-chick3
#| echo: false
#| eval: true

chick3 <- ChickWeight[ChickWeight$Diet == 1 | ChickWeight$Diet == 4, ]
head(chick3, 5)
  weight Time Chick Diet
1     42    0     1    1
2     51    2     1    1
3     59    4     1    1
4     64    6     1    1
5     76    8     1    1

Interpretación: chick3 combina todas las observaciones de las dietas 1 y 4 en un único data frame. Las primeras filas mostradas pertenecen a la dieta 1 porque el data frame original está ordenado por Chick y los pollitos de la dieta 1 aparecen primero. Este subconjunto es útil para comparar directamente el efecto de estas dos dietas, por ejemplo mediante un boxplot o un modelo lineal. El operador | es el equivalente en R base del filter(Diet %in% c(1, 4)) de dplyr.

#| label: res-chick4
#| echo: false
#| eval: true

chick4 <- ChickWeight[ChickWeight$weight > 100, ]
head(chick4, 5)
   weight Time Chick Diet
7     106   12     1    1
8     125   14     1    1
9     149   16     1    1
10    171   18     1    1
11    199   20     1    1

Interpretación: chick4 conserva únicamente las observaciones donde el pollito ya superó los 100 gramos de peso corporal. Las primeras filas del resultado corresponden a días de medición avanzados (días 8 a 12 aproximadamente), lo que indica que los pollitos tardan al menos una semana en alcanzar ese umbral de peso bajo condiciones normales. Este tipo de filtro es útil para analizar únicamente la fase de crecimiento acelerado del experimento.

#| label: res-chick5
#| echo: false
#| eval: true

chick5 <- ChickWeight[ChickWeight$Time == 10, 1:3]
head(chick5, 5)
   weight Time Chick
6      93   10     1
18    103   10     2
30     99   10     3
42     87   10     4
54    106   10     5

Interpretación: chick5 representa una “fotografía transversal” del experimento en el día 10: muestra el peso de cada pollito en ese momento específico, junto con su identificador, pero omite la variable Diet. Este tipo de subconjunto es útil para comparar el peso en un punto de tiempo fijo entre todos los animales. Si se quisiera incluir la dieta para identificar diferencias entre grupos, bastará con eliminar la restricción de columnas o incluir la posición 4 en el rango de selección.

#| label: res-iris1

iris1 <- iris[ , "Species"]

[1] setosa setosa setosa setosa setosa

Interpretación: El resultado de iris1 es un factor con tres niveles (setosa, versicolor, virginica), no un data frame. 3.4.3 dplyr con ChickWeight

#| label: res-chick1d
#| echo: false
#| eval: true

chick1_d <- ChickWeight %>% filter(Diet == 2)
head(chick1_d, 5)
  weight Time Chick Diet
1     40    0    21    2
2     50    2    21    2
3     62    4    21    2
4     86    6    21    2
5    125    8    21    2

Interpretación: El resultado de chick1_d es idéntico al de chick1 obtenido con R base: conserva todas las observaciones de la dieta 2 con sus cuatro columnas. La ventaja de filter() sobre la indexación con which() es sintáctica: no es necesario repetir el nombre del data frame dentro de la condición (no se escribe ChickWeight$Diet), lo que hace el código más corto y legible, especialmente en pipelines con múltiples filtros encadenados.

#| label: res-chick2d
#| echo: false
#| eval: true

chick2_d <- ChickWeight %>% filter(Chick == 4) %>% select(weight, Time)
head(chick2_d, 5)
  weight Time
1     42    0
2     49    2
3     56    4
4     67    6
5     74    8

Interpretación: chick2_d produce el mismo resultado que chick2 de R base: el historial de peso del pollito 4 con únicamente las columnas weight y Time. La cadena de verbos expresa la intención de forma secuencial y legible: “toma ChickWeight, filtra el pollito 4, luego quédate con peso y tiempo”. En R base, ambas operaciones se fusionan en una sola expresión de indexación [ , ], lo que puede resultar menos intuitivo al leer.

#| label: res-chick3d
#| echo: false
#| eval: true

chick3_d <- ChickWeight %>% filter(Diet == 1 | Diet == 4)
head(chick3_d, 5)
  weight Time Chick Diet
1     42    0     1    1
2     51    2     1    1
3     59    4     1    1
4     64    6     1    1
5     76    8     1    1

Interpretación: chick3_d es estructuralmente idéntico a chick3: contiene todas las observaciones de las dietas 1 y 4 con sus cuatro columnas. El operador %in% mencionado en el comentario es especialmente útil cuando se necesita filtrar por más de dos valores de la misma variable, ya que evita escribir múltiples condiciones con |. Por ejemplo, filter(Diet %in% c(1, 2, 4)) es más claro que filter(Diet == 1 | Diet == 2 | Diet == 4).

#| label: res-chick4d
#| echo: false
#| eval: true

chick4_d <- ChickWeight %>% filter(weight > 100)
head(chick4_d, 5)
  weight Time Chick Diet
1    106   12     1    1
2    125   14     1    1
3    149   16     1    1
4    171   18     1    1
5    199   20     1    1

3.5 Problemas Encontrados

  • La sintaxis de R base con [filas, columnas] resultó confusa al principio porque requiere recordar el orden (primero filas, luego columnas) y usar el signo $ para referirse a columnas por nombre dentro de las condiciones de filtrado.

  • El operador %in% no era conocido; fue necesario buscarlo en la documentación porque al intentar combinar muchos valores con | el código se volvía largo y propenso a errores tipográficos.

  • Hubo confusión inicial entre filter() (para filas) y select() (para columnas) en dplyr; en R base ambas operaciones se realizan con [ , ], lo que llevó en algunos intentos a usar los verbos en orden invertido.

  • Al seleccionar una sola columna en R base con [ , "Species"] se obtiene un vector en lugar de un data frame, lo que causó un error al intentar aplicarle directamente verbos de dplyr en pruebas posteriores. Con dplyr esto no ocurre porque select() siempre preserva la estructura de data frame.

  • El operador pipe %>% requirió práctica para interiorizar el flujo de datos: el objeto a la izquierda se convierte en el primer argumento de la función a la derecha. Una vez comprendido, los pipelines encadenados resultaron más legibles que las expresiones anidadas de R base.

3.6 Aprendizaje

  • R ofrece dos paradigmas equivalentes para manipular data frames: R base (más compacto pero menos legible para operaciones complejas) y dplyr (más expresivo y legible, especialmente en pipelines largos con múltiples pasos encadenados).

  • En dplyr, filter() opera sobre filas y select() opera sobre columnas; la separación explícita de estas operaciones en verbos distintos obliga a pensar con mayor claridad qué dimensión del data frame se está modificando.

  • El operador %in% es más eficiente y escalable que encadenar múltiples condiciones con | cuando se quieren incluir varios valores de una misma variable categórica.

  • La combinación group_by() + summarise() permite obtener estadísticas descriptivas por grupo en muy pocas líneas, algo que en R base requeriría tapply(), aggregate() o un bucle explícito.

  • Se confirmó que select() en dplyr siempre devuelve un data frame, mientras que [ , nombre] en R base devuelve un vector cuando se selecciona una sola columna. Esta diferencia es relevante cuando el resultado se usa como entrada para funciones que requieren un data frame.

  • Los problemas se resolvieron comparando sistemáticamente los resultados de R base y dplyr para verificar que fueran idénticos, y consultando la documentación con ?filter, ?select y ?which.

### R Base: Indexación de Data Frames

La sintaxis general de indexación en R es `data[filas, columnas]`. Dejar un espacio en blanco en alguna dimensión significa *conservar todo* en esa dimensión. El operador `[ ]` es el indexador fundamental de R para vectores, matrices y data frames; para data frames acepta dos argumentos separados por una coma, y omitir uno de los dos conserva esa dimensión completa.

```r

#| label: sintaxis-general
#| echo: true
#| eval: false

datos[filas, columnas]   # seleccionar filas y columnas específicas
datos[ , ]               # conservar todas las filas y todas las columnas

```

#### Selección de columnas

Las columnas de un data frame pueden seleccionarse por posición (número entero), por nombre (cadena de texto) o excluyéndolas con un índice negativo. El operador `:` genera una secuencia de enteros consecutivos, y `c()` construye vectores de índices no contiguos.

```r

#| label: seleccion-columnas
#| echo: true
#| eval: false

# ── Por posición individual ───────────────────────────────────────────────────

nuevo_dato <- datos[ , 40]

# ── Por posiciones no contiguas ───────────────────────────────────────────────

nuevo_dato <- datos[ , c(2, 5)]

# ── Por rango de posiciones contiguas ─────────────────────────────────────────

nuevo_dato <- datos[ , c(2:5)]

# ── Exclusión de una columna (signo negativo) ─────────────────────────────────

nuevo_dato <- datos[ , -5]

```

#### Selección de filas

Las filas pueden seleccionarse por rango de posición, por condición lógica usando `which()`, o con la función `subset()`. `which()` devuelve los índices numéricos de los elementos de un vector lógico que son `TRUE`; `subset()` ofrece una sintaxis más legible para filtrar y seleccionar simultáneamente.

```r

#| label: seleccion-filas
#| echo: true
#| eval: false

# ── Por rango de posición ─────────────────────────────────────────────────────

nuevo_dato <- datos[1:50, ]

# ── Por condición de texto con which() ───────────────────────────────────────

nuevo_dato <- datos[which(datos$gender == "mujer"), ]

# ── Condición combinada con AND lógico (&) ────────────────────────────────────

nuevo_dato <- datos[which(datos$gender == "mujer" & datos$age >= 65), ]

# ── Con subset(): filtrado de filas y selección de columnas en un paso ────────

nuevo_dato <- subset(datos, age >= 20, select = c(ID, Weight))

```

### Tidyverse: Manipulación con `dplyr`

`dplyr` es un paquete del ecosistema Tidyverse que ofrece verbos especializados para manipular data frames de forma legible. Las operaciones se encadenan con el operador pipe `%>%`, que pasa el resultado de una función como primer argumento de la siguiente función en la cadena. La instalación solo es necesaria una vez por entorno de trabajo; la carga con `library()` debe hacerse al inicio de cada sesión.

#### Verbos principales de `dplyr`

| Verbo          | Propósito                               |
|----------------|-----------------------------------------|
| `select()`     | Elegir columnas                         |
| `filter()`     | Conservar filas según una condición     |
| `mutate()`     | Crear o modificar columnas              |
| `arrange()`    | Ordenar filas                           |
| `relocate()`   | Reordenar columnas                      |
| `summarise()`  | Calcular resúmenes estadísticos         |
| `group_by()`   | Agrupar para operaciones por grupo      |

Se crea el objeto `df` como copia de `iris` para trabajar con los ejemplos sin modificar el data frame original.

```r

#| label: crear-df
#| echo: true
#| eval: false

df <- iris

```

#### `select()`

`select()` conserva únicamente las columnas especificadas. El signo negativo excluye una columna por nombre. `starts_with()` es una función auxiliar (*selection helper*) que selecciona todas las columnas cuyo nombre comienza con el prefijo indicado, lo que es útil cuando el data frame tiene muchas columnas con un patrón de nombrado común.

```r

#| label: select-ejemplos
#| echo: true
#| eval: false

# ── Por nombre ────────────────────────────────────────────────────────────────

df %>% select(Sepal.Length, Sepal.Width)

# ── Exclusión por nombre ──────────────────────────────────────────────────────

df %>% select(-Species)

# ── Por prefijo del nombre ────────────────────────────────────────────────────

df %>% select(starts_with("Sepal"))

```

#### `filter()`

`filter()` conserva únicamente las filas que cumplen la condición dada. `==` es el operador de igualdad (dos signos `=`; uno solo es asignación). `&` combina condiciones con AND lógico (ambas deben cumplirse), y `|` las combina con OR (basta con que se cumpla una).

```r

#| label: filter-ejemplos
#| echo: true
#| eval: false

# ── Igualdad de categoría ─────────────────────────────────────────────────────

df %>% filter(Species == "setosa")

# ── Condición numérica ────────────────────────────────────────────────────────

df %>% filter(Sepal.Length > 5)

# ── AND lógico (&) ────────────────────────────────────────────────────────────

df %>% filter(Species == "setosa" & Sepal.Length > 5)

# ── OR lógico (|) ─────────────────────────────────────────────────────────────

df %>% filter(Species == "setosa" | Species == "virginica")

```

#### `mutate()`

`mutate()` añade una nueva columna al data frame sin eliminar las existentes. El nombre de la nueva columna se define a la izquierda del `=` y la expresión de la derecha puede usar cualquier operación vectorizada de R.

```r

#| label: mutate-ejemplo
#| echo: true
#| eval: false

df %>% mutate(Sepal.Area = Sepal.Length * Sepal.Width)

```

#### `arrange()`

`arrange()` ordena las filas del data frame según la columna indicada. Por defecto el orden es ascendente; `desc()` invierte el orden a descendente y es una función auxiliar usada exclusivamente dentro de `arrange()`.

```r

#| label: arrange-ejemplo
#| echo: true
#| eval: false

# ── Orden ascendente (por defecto) ────────────────────────────────────────────

df %>% arrange(Sepal.Length)

# ── Orden descendente ─────────────────────────────────────────────────────────

df %>% arrange(desc(Sepal.Length))

```

#### `relocate()`

`relocate()` mueve una o más columnas a una posición distinta del data frame. Sin argumentos adicionales, mueve la columna al principio. El argumento `after =` la coloca inmediatamente después de la columna de referencia indicada.

```r

#| label: relocate-ejemplo
#| echo: true
#| eval: false

# ── Mover al principio ────────────────────────────────────────────────────────

df %>% relocate(Sepal.Length)

# ── Mover después de una columna específica ───────────────────────────────────

df %>% relocate(Sepal.Length, after = Species)

```

#### `summarise()`

`summarise()` colapsa todo el data frame en una sola fila que contiene las estadísticas calculadas. `mean()` calcula el promedio aritmético de un vector numérico. El resultado es un data frame de 1 fila × 1 columna.

```r

#| label: summarise-ejemplo
#| echo: true
#| eval: false

df %>% summarise(media_largo = mean(Sepal.Length))

```

#### `group_by()` + `summarise()`

`group_by()` no modifica los datos visualmente, pero añade metadata interna que indica a `summarise()` que las operaciones deben realizarse de forma independiente dentro de cada nivel del factor. El resultado es un data frame de tantas filas como grupos existan.

```r

#| label: group-summarise
#| echo: true
#| eval: false

df %>%
  group_by(Species) %>%
  summarise(media_largo = mean(Sepal.Length))

```

#### Pipeline encadenado — ejemplo completo

Este pipeline combina tres verbos en secuencia:

1. `filter()`: conserva solo las flores de la especie setosa.
2. `mutate()`: añade la columna Sepal.Area = largo × ancho del sépalo.
3. `group_by()` + `summarise()`: calculan la media del área para esa especie.

El resultado es un data frame de 1 fila × 2 columnas (Species, area_media). El operador %>% evita guardar objetos intermedios y hace el flujo legible de arriba hacia abajo, como una receta de pasos secuenciales.

```r

#| label: pipeline-encadenado
#| echo: true
#| eval: false

df %>%
  filter(Species == "setosa") %>%
  mutate(Sepal.Area = Sepal.Length * Sepal.Width) %>%
  group_by(Species) %>%
  summarise(area_media = mean(Sepal.Area))

```

Las primeras filas del resultado corresponden a días de medición avanzados (días 8 a 12 aproximadamente), lo que indica que los pollitos tardan al menos una semana en alcanzar ese umbral de peso bajo condiciones normales. Este tipo de filtro es útil para analizar únicamente la fase de crecimiento acelerado del experimento.

```{r}
#| label: res-chick5
#| echo: false
#| eval: true

chick5 <- ChickWeight[ChickWeight$Time == 10, 1:3]
head(chick5, 5)

```

**Interpretación:** `chick5` representa una "fotografía transversal" del experimento en el día 10: muestra el peso de cada pollito en ese momento específico, junto con su identificador, pero omite la variable `Diet`. Este tipo de subconjunto es útil para comparar el peso en un punto de tiempo fijo entre todos los animales. Si se quisiera incluir la dieta para identificar diferencias entre grupos, bastará con eliminar la restricción de columnas o incluir la posición 4 en el rango de selección.

### R Base con `iris`

```{r}
#| label: res-iris1
#| echo: false
#| eval: true

iris1 <- iris[ , "Species"]
head(iris1, 5)

```

**Interpretación:** El resultado de `iris1` es un **factor** con tres niveles (`setosa`, `versicolor`, `virginica`), no un data frame. R imprime las primeras cinco observaciones seguidas de la línea `Levels: setosa versicolor virginica`, que indica los tres valores posibles del factor. Las cinco primeras filas corresponden a *setosa* porque el data frame original está ordenado con las 50 observaciones de cada especie en bloques consecutivos. Esta diferencia de tipo (vector vs. data frame) es un punto de confusión frecuente entre R base y `dplyr`, donde `select()` siempre preserva la estructura de data frame.

```{r}
#| label: res-iris2
#| echo: false
#| eval: true

iris2 <- iris[ , -5]
head(iris2, 5)

**Interpretación:** `iris2` es un data frame de 150 filas y 4 columnas numéricas (`Sepal.Length`, `Sepal.Width`, `Petal.Length`, `Petal.Width`), con la columna categórica `Species` eliminada. Este subconjunto es el formato requerido, por ejemplo, para calcular una matriz de correlaciones con `cor()` o para ajustar un modelo de componentes principales con `prcomp()`, ya que ambas funciones requieren entrada exclusivamente numérica.

```{r}
#| label: res-iris3
#| echo: false
#| eval: true

iris3 <- iris[iris$Sepal.Length > 5.0, ]
head(iris3, 5)

```

**Interpretación:** `iris3` contiene todas las flores con sépalo largo mayor a 5.0 cm. De las 150 observaciones originales, 118 cumplen esta condición (solo 32 tienen sépalo ≤ 5.0 cm). La mayoría de las flores excluidas son de la especie *setosa*, que tiene sépalos sistemáticamente más cortos, con una media de 5.006 cm. Las primeras filas del resultado pertenecen a *setosa* con valores justo por encima del umbral (5.1 cm), lo que confirma que el filtro es inclusivo para valores mayores estrictos.

iris4 <- iris[which(iris$Species == "setosa"), ]

**Interpretación:** `iris4` contiene exactamente las 50 observaciones de la especie *setosa* con sus cinco columnas originales. Todas las filas de la columna `Species` muestran el valor `setosa`, confirmando que el filtro es correcto. Este subconjunto resulta útil para analizar las características morfológicas de la especie de forma aislada, por ejemplo para calcular sus estadísticos descriptivos o ajustar un modelo de regresión exclusivo para ese grupo.

#| label: res-iris5

Este subconjunto es relevante para análisis discriminantes o de agrupamiento donde se quiere estudiar la separabilidad entre las dos especies de mayor tamaño, que son morfológicamente más similares entre sí que con *setosa*.

### `dplyr` con `ChickWeight`

```{r}
#| label: res-chick1d
#| echo: false
#| eval: true

chick1_d <- ChickWeight %>% filter(Diet == 2)
head(chick1_d, 5)

```

**Interpretación:** El resultado de `chick1_d` es idéntico al de `chick1` obtenido con R base: conserva todas las observaciones de la dieta 2 con sus cuatro columnas. La ventaja de `filter()` sobre la indexación con `which()` es sintáctica: no es necesario repetir el nombre del data frame dentro de la condición (no se escribe `ChickWeight$Diet`), lo que hace el código más corto y legible, especialmente en pipelines con múltiples filtros encadenados.

```{r}
#| label: res-chick2d
#| echo: false
#| eval: true

chick2_d <- ChickWeight %>% filter(Chick == 4) %>% select(weight, Time)
head(chick2_d, 5)

```

**Interpretación:** `chick2_d` produce el mismo resultado que `chick2` de R base: el historial de peso del pollito 4 con únicamente las columnas `weight` y `Time`. La cadena de verbos expresa la intención de forma secuencial y legible: "toma `ChickWeight`, filtra el pollito 4, luego quédate con peso y tiempo". En R base, ambas operaciones se fusionan en una sola expresión de indexación `[ , ]`, lo que puede resultar menos intuitivo al leer.

```{r}
#| label: res-chick3d
#| echo: false
#| eval: true

chick3_d <- ChickWeight %>% filter(Diet == 1 | Diet == 4)
head(chick3_d, 5)

```

**Interpretación:** `chick3_d` es estructuralmente idéntico a `chick3`: contiene todas las observaciones de las dietas 1 y 4 con sus cuatro columnas. El operador `%in%` mencionado en el comentario es especialmente útil cuando se necesita filtrar por más de dos valores de la misma variable, ya que evita escribir múltiples condiciones con `|`. Por ejemplo, `filter(Diet %in% c(1, 2, 4))` es más claro que `filter(Diet == 1 | Diet == 2 | Diet == 4)`.

```{r}
#| label: res-chick4d
#| echo: false
#| eval: true

chick4_d <- ChickWeight %>% filter(weight > 100)
head(chick4_d, 5)

```

**Interpretación:** `chick4_d` coincide exactamente con `chick4` de R base. Ambos devuelven las mismas filas y columnas porque la lógica del filtro es idéntica. La diferencia es únicamente estilística: `dplyr` evita el uso de `$` y hace el código más limpio. En análisis exploratorios donde se prueba rápidamente con diferentes umbrales de peso, la sintaxis de `dplyr` facilita modificar el valor 100 sin tener que localizar y editar múltiples referencias al nombre del data frame.

```{r}
#| label: res-chick5d
#| echo: false
#| eval: true

chick5_d <- ChickWeight %>% filter(Time == 10) %>% select(1:3)
head(chick5_d, 5)

```

**Interpretación:** `chick5_d` produce el mismo resultado que `chick5` de R base: la instantánea del día 10 con las columnas `weight`, `Time` y `Chick`. En `dplyr`, `select(1:3)` funciona de manera equivalente a `[ , 1:3]`, aunque dentro del paradigma Tidyverse es preferible seleccionar por nombre (`select(weight, Time, Chick)`) para que el código sea robusto frente a cambios en el orden de las columnas.

R lo imprime con encabezado de columna (`Species`), mientras que el vector de R base se imprime sin encabezado y con la línea de niveles al final. ### Resumen estadístico por especie

```{r}
#| label: res-summarise
#| echo: false
#| eval: true

iris %>%
  group_by(Species) %>%
  summarise(
    media_largo_sepalo = mean(Sepal.Length),
    media_ancho_sepalo = mean(Sepal.Width),
    media_largo_petalo = mean(Petal.Length),
    media_ancho_petalo = mean(Petal.Width)
  )
```

**Interpretación:** La tabla de resumen condensa 150 observaciones en tres filas, una por especie, mostrando los promedios de cada variable morfológica. Los patrones más destacados son los siguientes. *Setosa* tiene los pétalos más pequeños (largo medio ≈ 1.46 cm, ancho medio ≈ 0.25 cm), muy por debajo de las otras dos especies, lo que la hace claramente separable en análisis multivariados. *Virginica* presenta los pétalos más grandes (largo medio ≈ 5.55 cm, ancho medio ≈ 2.03 cm), lo que la diferencia bien de *setosa* pero con más solapamiento respecto a *versicolor* (largo medio ≈ 4.26 cm). En cuanto a los sépalos, *setosa* tiene el ancho medio mayor (≈ 3.43 cm) pero la longitud más corta (≈ 5.006 cm), lo que sugiere sépalos más anchos y cortos comparados con las otras dos especies. *Versicolor* y *virginica* muestran valores intermedios y similares en las variables de sépalo, lo que anticipa que serán las más difíciles de separar en un análisis discriminante basado únicamente en esas dos variables.

## Problemas Encontrados

- La sintaxis de R base con `[filas, columnas]` resultó confusa al principio porque requiere recordar el orden (primero filas, luego columnas) y usar el signo `$` para referirse a columnas por nombre dentro de las condiciones de filtrado.

- El operador `%in%` no era conocido; fue necesario buscarlo en la documentación porque al intentar combinar muchos valores con `|` el código se volvía largo y propenso a errores tipográficos.

- Hubo confusión inicial entre `filter()` (para filas) y `select()` (para columnas) en `dplyr`; en R base ambas operaciones se realizan con `[ , ]`, lo que llevó en algunos intentos a usar los verbos en orden invertido.

- Al seleccionar una sola columna en R base con `[ , "Species"]` se obtiene un vector en lugar de un data frame, lo que causó un error al intentar aplicarle directamente verbos de `dplyr` en pruebas posteriores. Con `dplyr` esto no ocurre porque `select()` siempre preserva la estructura de data frame.

- El operador pipe `%>%` requirió práctica para interiorizar el flujo de datos: el objeto a la izquierda se convierte en el primer argumento de la función a la derecha. Una vez comprendido, los pipelines encadenados resultaron más legibles que las expresiones anidadas de R base.

## Aprendizaje

- R ofrece dos paradigmas equivalentes para manipular data frames: R base (más compacto pero menos legible para operaciones complejas) y `dplyr` (más expresivo y legible, especialmente en pipelines largos con múltiples pasos encadenados).

- En `dplyr`, `filter()` opera sobre filas y `select()` opera sobre columnas; la separación explícita de estas operaciones en verbos distintos obliga a pensar con mayor claridad qué dimensión del data frame se está modificando.

- El operador `%in%` es más eficiente y escalable que encadenar múltiples condiciones con `|` cuando se quieren incluir varios valores de una misma variable categórica.

- La combinación `group_by()` + `summarise()` permite obtener estadísticas descriptivas por grupo en muy pocas líneas, algo que en R base requeriría `tapply()`, `aggregate()` o un bucle explícito.

- Se confirmó que `select()` en `dplyr` siempre devuelve un data frame, mientras que `[ , nombre]` en R base devuelve un vector cuando se selecciona una sola columna. Esta diferencia es relevante cuando el resultado se usa como entrada para funciones que requieren un data frame.

- Los problemas se resolvieron comparando sistemáticamente los resultados de R base y `dplyr` para verificar que fueran idénticos, y consultando la documentación con `?filter`, `?select` y `?which`.
