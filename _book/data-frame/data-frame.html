Indexación y Manipulación con R Base y dplyr

Author

4.1 Objetivo

Extraer subconjuntos de un dataframe de dos maneras distintas: usando la sintaxis de indexación de R base ([ , ], which(), subset()) y usando los verbos del paquete dplyr del ecosistema Tidyverse (filter(), select(), mutate(), arrange(), summarise(), etc.).

4.2 Datos utilizados

Se utilizaron dos bases de datos integradas en R base.

  • Descripción general
  • Código R
  • iris
  • ChickWeight

La base de datos utilizó iris incluida en R. Proviene de las observaciones de Ronal Fisher (1936) del género Iris. La variable categórica es la Species (setosa, versicolor y virginica) y la variable de repsuesta contiene Sepal.Length (longitud de sépalo), Sepal.Width (ancho de sépalo), Petal.Length (longitud de pétalo) y Petal.Width (ancho de pétalo).

La base de datos ChickWeight está incluida en R. Proviene de un experimento nutricional descrito por Crowder & Hand (1990) en el que se evaluó el efecto de cuatro dietas proteicas sobre el crecimiento de pollitos. La variable de respuesta es weight (peso corporal en gramos), la variable temporal es Time (día de medición, de 0 a 21), el identificador del sujeto es Chick (pollito 1 al 50) y la variable de agrupación es Diet (dieta asignada: 1, 2, 3 ó 4).

4.5 Problemas Encontrados

  • La sintaxis de R base con [filas, columnas] resultó confusa al principio porque requiere recordar el orden (primero filas, luego columnas) y usar el signo $ para referirse a columnas por nombre dentro de condiciones.
  • El operador %in% no era conocido; fue necesario buscarlo en la documentación porque al intentar combinar muchos valores con | el código se volvía largo y propenso a errores.
  • Hubo confusión entre filter() (para filas) y select() (para columnas) en dplyr; en R base ambas operaciones se hacen con [ , ], lo que al principio llevó a usarlos al revés.
  • Al usar select() con una sola columna en R base ([ , "Species"]) se obtuvo un vector en lugar de un data frame, lo que causó un error al intentar aplicarle head() en algunas pruebas. Con dplyr esto no ocurre porque select() siempre devuelve un data frame.
  • El operador pipe %>% requirió práctica para entender el flujo: el objeto a la izquierda se convierte en el primer argumento de la función a la derecha. Una vez interiorizado, el código encadenado resultó mucho más legible.

5 Aprendizaje

  • Se aprendió que R ofrece dos paradigmas equivalentes para manipular data frames: R base (más compacto pero menos legible) y dplyr (más expresivo y legible, especialmente en pipelines largos).
  • Se comprendió la diferencia entre seleccionar filas y columnas, y que en dplyr estas son operaciones distintas (filter() vs. select()), lo que obliga a pensar con más claridad qué se quiere hacer.
  • El operador %in% es más eficiente que encadenar múltiples condiciones con | cuando se quieren incluir varios valores de una misma variable.
  • group_by() + summarise() es una combinación muy poderosa para obtener estadísticas descriptivas por grupo en pocas líneas, algo que en R base requeriría un tapply() o un bucle.
  • Se reforzó el concepto de que en R los objetos se sobreescriben cuando se les asigna un nuevo valor, por lo que es importante nombrar los subconjuntos con nombres distintos para no perder los datos originales.
  • En general, los problemas se resolvieron consultando la documentación de dplyr con ?filter y ?select, y comparando los resultados de R base y dplyr para verificar que fueran idénticos.
